# Генеричке методе са више параметара

Генеричке методе могу да прихвате и више параметара. У зависности од ситуације,
ти параметри могу се дефинисати као параметри истог типа или различитих типова.

## Генеричке методе са више параметара истог типа

Један од примера у којима је потребно дефинисати више параметара истог типа
може да буде метода за замену вредности две променљиве:

```cs
public static void Zameni<T>(ref T a, ref T b)
{
    T temp = a;
    a = b;
    b = temp;
}
```

Методи `Zameni<T>()` могу се проследити два аргумента било ког типа, где се
типски параметар `T` одређује у тренутку позива методе. Након извршења следећег
програма...

```cs
static void Main()
{
    int a = 1, b = 2;
    double c = 3.4, d = 5.6;
    string e = "sedam", f = "osam";
    Zameni(ref a, ref b);
    Zameni(ref c, ref d);
    Zameni(ref e, ref f);
    Console.WriteLine("{0} {1}", a, b);
    Console.WriteLine("{0} {1}", c, d);
    Console.WriteLine("{0} {1}", e, f);
}
```

...у конзоли ће се исписати:

```text
2 1
5.6 3.4
osam sedam
```

## Генеричке методе са параметрима различитих типова

Генеричке методе могу да прихвате и више параметара различитих типова, где се у
угластим заградама дефинише више типских параметара `<T1, T2, ...>`.
Једноставан пример била би метода за испис три прослеђене вредности које су
различитих типова података чији типови нису унапред познати. Дефиниција методе
може да изгледа овако:

```cs
public static void IspisiVrednosti<T1, T2, T3>(T1 v1, T2 v2, T3 v3)
{
    Console.WriteLine("{0}, {1}, {2}", v1, v2, v3);
}
```

Генеричка метода `IspisiVrednosti` прима три параметра произвољних типова који
се одређују аутоматски на основу прослеђених аргумената. Типови могу бити било
који типови података, укључујући вредносне, референтне и кориснички дефинисане
типове:

```cs
static void Main()
{
    IspisiVrednosti(1, 2.3, Math.PI);
    IspisiVrednosti('C', "Pozdrav!", DateTime.Now);
    IspisiVrednosti(sizeof(double), false, !false);
}
```

Након извршења овог програма у конзоли ће се исписати...

```text
1, 2.3, 3.14159265358979
C, Pozdrav!, 4/3/2025 11:10:09 AM
8, False, True
```

У већини случајева није потребно експлицитно наводити типске параметре приликом
позива генеричке методе, јер ће компајлер аутоматски из аргумената закључити
типове. Ова појава се назива инференција (закључивање) типова.

Нека је задатак да дефинишеш генеричку методу са три параметра (који могу бити
различитих типова) која враћа стринг сачињен од прослеђених аргумената.

```cs
public static string SpojiTriVrednosti<T1, T2, T3>(T1 v1, T2 v2, T3 v3)
{
    return $"{v1} {v2} {v3}";
}
```

Генеричка метода `SpojiTriVrednosti` прима три параметра произвољних типова,
који се одређују аутоматски на основу прослеђених аргумената. У телу методе
користи се интерполација стрингова да би се прослеђени аргументи претворили у
стринг.

```cs
static void Main()
{
    string res = SpojiTriVrednosti(1, 2.2, "tri");
    Console.WriteLine(res);
    res = SpojiTriVrednosti('C', "Pozdrav!", !false);
    Console.WriteLine(res);
}
```

Након извршења овог програма у конзоли ће се исписати:

```text
1 2.2 tri
C Pozdrav! True
```

## Генеричке методе са променљивим бројем параметара

Ако метода треба да прими променљиви број параметара различитих типова, може се
користити `params object[]`. На пример, нека је задатак да дефинишеш генеричку
методу са непознатим бројем параметара који могу бити различитих типова, која
враћа стринг сачињен од прослеђених аргумената.

```cs
public static string SpojiVrednosti(params object[] parametri)
{
    return string.Join(" ", parametri);
}
```

У телу методе користи се `string.Join` да би се комбиновали сви параметри у један
стринг, где је сепаратор један размак. Овако метода може да прими било који
број параметара било ког типа, где се сви параметри третирају као `object`:

```cs
static void Main()
{
    string res = SpojiVrednosti(1, 2.2, "tri", 'C', "Pozdrav!");
    Console.WriteLine(res);
}
```

Након извршења овог програма у конзоли ће се исписати:

```text
1 2.2 tri C Pozdrav!
```

Претходна метода `SpojiTriVrednosti` је идеална за спајање унапред познатог
броја прослеђених аргумената у стринг, док је метода `SpojiVrednosti`
флексибилнија и подржава спајање било ког броја параметара. Обе методе су
генеричке и раде са било којим типовима података, укључујући вредносне,
референтне и кориснички дефинисане типове.
